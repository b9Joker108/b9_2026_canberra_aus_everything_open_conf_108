**1. Layered Documentation Structure**

**Best practice on GitHub is to use an explicit, layered set of markdown files in the `.github/` folder and/or repo root:**

- `CODE_OF_CONDUCT.md`
- `CONTRIBUTING.md`
- `.github/CULTURE.md` (or another clear name reflecting your custom culture and values)
- Optionally: `.github/STANDARDS.md`, `.github/VALUES.md`, or a combined ‚Äúteam manual‚Äù

These can be explicitly interlinked and cross-referenced, making the workflow and expectations crystal clear for all humans and automation.

---

**2. Gold-Standard Integration & Referencing**

**How to integrate your requirements:**
- In `CONTRIBUTING.md`, reference and point to both the official GitHub docs and your repo‚Äôs own code of conduct, culture, and standards files at the start (i.e. ‚ÄúBy contributing, you agree to follow our [Code of Conduct](CODE_OF_CONDUCT.md) and [Cultural Standards](.github/CULTURE.md)‚Äù).

**3. Tone and Enforcement**

1. Create a script (e.g., countdown.py) that calculates days remaining:

   ```yaml

   jobs:

       runs-on: ubuntu-latest

         - uses: actions/checkout@v3

             python-version: '3.x'
         - run: python countdown.py
         - run: |
             git config --global user.name 'github-actions[bot]'
             git config --global user.email 'github-actions[bot]@users.noreply.github.com'
             git add README.md
             git commit -m "Update countdown"
             git push
   ```
